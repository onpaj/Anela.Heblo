## Summary

`POST InvoiceClassification/ClassifyInvoices` is slow (~15s) when processing multiple invoices because the handler fetches each invoice individually in a sequential `foreach` loop — one Flexi API call per invoice.

## App Insights Evidence

- Slow requests observed in last 24h (avg ~15s)
- Endpoint: `POST InvoiceClassification/ClassifyInvoices`

## Root Cause

In `ClassifyInvoicesHandler.cs`, when `request.InvoiceIds` is provided, each invoice is fetched via a separate `await _invoicesClient.GetInvoiceByIdAsync(invoiceId)` call inside a `foreach` loop. With N invoices, this results in N sequential round-trips to the Flexi API.

```csharp
foreach (var invoiceId in request.InvoiceIds)
{
    var invoice = await _invoicesClient.GetInvoiceByIdAsync(invoiceId);
    ...
}
```

## Proposed Fix

1. Batch the individual invoice fetches using `Task.WhenAll` to parallelize them:
   ```csharp
   var tasks = request.InvoiceIds.Select(id => _invoicesClient.GetInvoiceByIdAsync(id));
   var invoices = await Task.WhenAll(tasks);
   ```
2. Alternatively, add a `GetInvoicesByIdsAsync(IEnumerable<int>)` method to `IReceivedInvoicesClient` that fetches all in one Flexi API call if the API supports bulk fetch.
