# Implementation: LogisticsController User-ID Resolution Cleanup

## What was implemented

Delete-and-simplify refactor that removes dead GUID-parsing code from `LogisticsController`. The sentinel GUID `00000000-0000-0000-0000-000000000001` and the `request.UserId` assignment pattern were confirmed to have zero readers anywhere in the Logistics application/domain/persistence layers — the actual audit field (`GiftPackageManufactureLog.CreatedBy`, a string) is already populated by `GiftPackageManufactureService` via its own `ICurrentUserService.GetCurrentUser().Name` call. The refactor deletes rather than relocates.

## Files created/modified

- `backend/src/Anela.Heblo.API/Controllers/LogisticsController.cs` — removed `ICurrentUserService` field/constructor param/using, simplified `CreateGiftPackageManufacture` and `DisassembleGiftPackage` to one-liner mediator sends
- `backend/src/Anela.Heblo.Application/Features/Logistics/UseCases/GiftPackageManufacture/UseCases/CreateGiftPackageManufacture/CreateGiftPackageManufactureRequest.cs` — removed `public Guid UserId { get; set; }`
- `backend/src/Anela.Heblo.Application/Features/Logistics/UseCases/GiftPackageManufacture/UseCases/DisassembleGiftPackage/DisassembleGiftPackageRequest.cs` — removed `public Guid UserId { get; set; }`
- `frontend/src/api/generated/api-client.ts` — regenerated by `dotnet build`; `userId` dropped from both request interfaces/classes
- `frontend/src/components/pages/GiftPackageManufacturing/index.tsx` — removed `userId` field from `handleManufacture` request literal
- `frontend/src/api/hooks/__tests__/useGiftPackageManufacturing.test.ts` — removed `userId: "user123"` from 4 Jest request literals
- `memory/decisions/logistics-no-userid-resolver.md` — new; records why no `UserIdResolver` helper was created and what pattern to use if a GUID audit column is ever needed

## Notes

The arch review determined FR-2 (handler-level resolution) and FR-3 (UserIdResolver helper) from the spec were unnecessary — `request.UserId` had zero consumers. The spec was amended before implementation. The `mockManufactureResponse.data.userId` in the test file was intentionally left alone (pre-existing loose response mock, out of scope per surgical-change rule). `DashboardController` has the same dead-code pattern and is explicitly out of scope.

## PR Summary

Deleted a dead user-ID GUID resolution block that lived in two LogisticsController action methods, wrote to `request.UserId`, and was never read by any handler, service, or persistence column. The audit string on `GiftPackageManufactureLog.CreatedBy` was and remains populated by `GiftPackageManufactureService` directly. The sentinel literal `00000000-0000-0000-0000-000000000001` is removed from the codebase entirely. Both action methods are now one-liner mediator sends, matching the shape of every other action in the controller. The `UserId` DTO properties and their downstream TypeScript client types are also removed, with the frontend call site and Jest mocks updated to match.
