# Design: Bank Frontend Hook Redefines DTO Types

## Component Design

### `frontend/src/api/hooks/useBankStatements.ts`

**Responsibility:** Expose React Query hooks for bank statement operations. All HTTP transport is delegated to the generated `apiClient`; this file owns only query key management, parameter mapping, and view-model projection.

**Interfaces to delete** (duplicates of generated types):
- `interface BankStatementImportDto`
- `interface GetBankStatementListResponse`
- `interface BankStatementImportResult`
- `interface BankImportResponse`
- `interface BankAccountDto`

**Interfaces to keep** (hook-layer adapters, not DTO duplicates):
- `export interface GetBankStatementListRequest` — maps caller-friendly fields to the 12 positional parameters expected by `bankStatements_GetBankStatements`
- `export interface BankImportRequest` — accepts `dateFrom`/`dateTo` as strings; the hook converts them to `Date` before constructing `BankImportRequestDto`
- `export interface AccountOption` — view-model projected from `BankAccountDto` for UI consumption
- `export interface GetBankStatementImportStatisticsRequest` — already correct, no change

**Imports to add:**
```typescript
import {
  GetBankStatementListResponse,
  BankStatementImportDto,
  BankStatementImportResultDto,
  BankAccountDto,
  BankImportRequestDto,
} from '../generated/api-client';
```

**Re-exports to add** (consumers reference these types through the hook module):
```typescript
export type { GetBankStatementListResponse, BankStatementImportDto, BankStatementImportResultDto };
```

**Hook method boundaries:**

| Hook | Generated client call | Return type |
|---|---|---|
| `useBankStatementsList` | `apiClient.bankStatements_GetBankStatements(...12 params)` | `GetBankStatementListResponse` |
| `useBankStatementImport` | `apiClient.bankStatements_ImportStatements(dto: BankImportRequestDto)` | `BankStatementImportResultDto` |
| `useBankStatementAccounts` | `apiClient.bankStatements_GetAccounts()` | `AccountOption[]` (projected) |

### `frontend/src/api/hooks/__tests__/useBankStatements.test.ts`

**Responsibility:** Unit-test the three hooks in isolation by mocking generated client methods directly.

**Changes required:**
- Remove mock pattern `(apiClient as any).http.fetch` — this reached into the private NSwag transport layer.
- Replace with typed method mocks, e.g.:
  ```typescript
  jest.spyOn(apiClient, 'bankStatements_GetBankStatements').mockResolvedValue(...);
  jest.spyOn(apiClient, 'bankStatements_ImportStatements').mockResolvedValue(...);
  jest.spyOn(apiClient, 'bankStatements_GetAccounts').mockResolvedValue(...);
  ```
- Add test cases for `useBankStatementsList` and `useBankStatementImport` (currently untested).

### Hook-layer adapter shapes (kept local, not DTO duplicates)

**`GetBankStatementListRequest`** — caller input for `useBankStatementsList`; maps to the 12 positional parameters of `bankStatements_GetBankStatements`.

**`BankImportRequest`** — caller input for `useBankStatementImport`; uses string dates for convenience; hook converts to `Date` before constructing `BankImportRequestDto`.

**`AccountOption`** — view-model projected from `BankAccountDto[]` returned by `bankStatements_GetAccounts`.

```typescript
export interface AccountOption {
  value: string;
  label: string;
}
```
