## PR Summary

Freed `ManufactureStockTakingController` from its silent compile-time dependency on the Catalog module by introducing a Manufacture-owned `GetManufactureStockTakingHistory` use case. The new handler follows the same pattern as `SubmitManufactureStockTakingHandler` — reading `CatalogAggregate.StockTakingHistory` via the existing `IManufactureCatalogSource` cross-module contract, sorting/paging in-memory, and projecting to a Manufacture-owned DTO. The HTTP endpoint URL, query parameters, and JSON payload are identical to before. The TypeScript client was regenerated and the frontend hook's imported type updated. The Catalog endpoint at `/api/StockTaking/history` and all Catalog-side code are unaffected.
