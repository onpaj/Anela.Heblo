# Photobank `GET /api/photobank/photos` Performance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring `GET /api/photobank/photos` from a 13.5 s average response time to p50 ≤ 200 ms / p95 ≤ 800 ms by rewriting `PhotobankRepository.BuildFilterQuery` / `GetPhotosAsync` to be sargable, tracking-free, and split-loaded, and by adding three supporting PostgreSQL indexes (composite B-tree on `(ModifiedAt DESC, Id DESC)`, GIN trigram on `LOWER(FolderPath || '/' || FileName)`, B-tree on `PhotoTags(TagId, PhotoId)`).

**Architecture:** The change is confined to a single Vertical Slice (`Features/Photobank`) and one EF Core migration. The public contract — controller, request DTO, response DTO, OpenAPI shape, error codes — is unchanged. The repository internally switches from `Contains` to `EF.Functions.Like(LOWER(...), $"%{escaped}%", "\\")` (reusing the `LeafletRepository.cs:173` escape pattern), replaces N stacked `EXISTS` subqueries with a single `GROUP BY ... HAVING COUNT(DISTINCT TagId) = N` subquery, applies `AsNoTracking()` + `AsSplitQuery()`, and adds `.ThenByDescending(p => p.Id)` as a deterministic pagination tiebreaker. The existing `PhotobankRepositoryFilterTests` move from EF Core InMemory to PostgreSQL Testcontainers (InMemory cannot translate `EF.Functions.Like`) and a new Testcontainers-based performance regression test (FR-8) seeds 10 000 photos to guard the SLO.

**Tech Stack:** .NET 8, EF Core 8 (Npgsql provider), PostgreSQL with `pg_trgm` extension, MediatR, xUnit, FluentAssertions, Moq, Testcontainers (`Testcontainers.PostgreSql` 3.6.0 — already in `Anela.Heblo.Tests.csproj`).

**Spec amendments locked in (per `arch-review.r1.md`):**
- `PhotobankRepositoryFilterTests` / `PhotobankRepositoryRegexFilterTests` / `PhotobankRepositoryPathRegexFilterTests` are migrated from EF InMemory to PostgreSQL Testcontainers. Assertions are unchanged — only the infrastructure swaps.
- Pagination order becomes `OrderByDescending(p => p.ModifiedAt).ThenByDescending(p => p.Id)`. The supporting B-tree index is composite descending on both columns.
- `search` values that are null / empty / whitespace-only continue to be treated as "no filter" — `Trim()` before deciding.
- Migration runs `CREATE EXTENSION IF NOT EXISTS pg_trgm` then the GIN index. `CREATE INDEX CONCURRENTLY` is not required (low write rate on `Photos`); the index is created inside the default migration transaction.
- `BuildFilterQuery` is also called by `CountFilteredPhotosAsync` and `GetFilteredPhotoIdsMissingTagAsync`. The refactor must keep their result sets identical — regression-tested in Task 5.
- Index for sort is **composite**, declarative: `HasIndex(x => new { x.ModifiedAt, x.Id }).IsDescending()`. The GIN trigram index is raw SQL only (EF cannot model GIN on an expression key).
- Predicate uses `EF.Functions.Like((p.FolderPath + "/" + p.FileName).ToLower(), $"%{escaped}%", "\\")` so the planner matches the indexed expression `LOWER("FolderPath" || '/' || "FileName")` byte-for-byte.

```csharp
using Anela.Heblo.Domain.Features.Photobank;
using Anela.Heblo.Persistence.Extensions;
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Metadata.Builders;

namespace Anela.Heblo.Persistence.Photobank
{
    public class PhotoConfiguration : IEntityTypeConfiguration<Photo>
    {
        public void Configure(EntityTypeBuilder<Photo> builder)
        {
            builder.ToTable("Photos", "public");
            builder.HasKey(x => x.Id);
            builder.Property(x => x.SharePointFileId).HasMaxLength(500).IsRequired();
            builder.HasIndex(x => x.SharePointFileId).IsUnique().HasDatabaseName("IX_Photos_SharePointFileId");
            builder.Property(x => x.FolderPath).HasMaxLength(2000).IsRequired();
            builder.Property(x => x.FileName).HasMaxLength(200).IsRequired();
            builder.Property(x => x.SharePointWebUrl).HasMaxLength(2000);
            builder.Property(x => x.DriveId).HasMaxLength(500);
            builder.Property(x => x.MimeType).HasMaxLength(50);
            builder.Property(x => x.IndexedAt).IsRequired().AsUtcTimestamp();
            builder.Property(x => x.ModifiedAt).IsRequired().AsUtcTimestamp();
            builder.HasIndex(x => x.FolderPath).HasDatabaseName("IX_Photos_FolderPath");
            builder.HasIndex(x => new { x.ModifiedAt, x.Id })
                .IsDescending(true, true)
                .HasDatabaseName("IX_Photos_ModifiedAt_Id");
            builder.HasMany(x => x.Tags)
                .WithOne(x => x.Photo)
                .HasForeignKey(x => x.PhotoId)
                .OnDelete(DeleteBehavior.Cascade);
        }
    }
}
```

```csharp
using Anela.Heblo.Domain.Features.Photobank;
using Anela.Heblo.Persistence.Extensions;
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Metadata.Builders;

namespace Anela.Heblo.Persistence.Photobank
{
    public class PhotoTagConfiguration : IEntityTypeConfiguration<PhotoTag>
    {
        public void Configure(EntityTypeBuilder<PhotoTag> builder)
        {
            builder.ToTable("PhotoTags", "public");
            builder.HasKey(x => new { x.PhotoId, x.TagId });
            builder.Property(x => x.Source).HasConversion<string>().HasMaxLength(20).IsRequired();
            builder.Property(x => x.CreatedAt).IsRequired().AsUtcTimestamp();
            builder.HasIndex(x => new { x.TagId, x.PhotoId })
                .HasDatabaseName("IX_PhotoTags_TagId_PhotoId");
            builder.HasOne(x => x.Tag)
                .WithMany(x => x.PhotoTags)
                .HasForeignKey(x => x.TagId)
                .OnDelete(DeleteBehavior.Cascade);
        }
    }
}
```

```csharp
        private static string EscapeLikePattern(string input) =>
            input.Replace("\\", "\\\\").Replace("%", "\\%").Replace("_", "\\_");

        private IQueryable<Photo> BuildFilterQuery(List<string>? tags, string? search, bool useRegex)
        {
            var query = _context.Photos.AsNoTracking().AsQueryable();

            if (!string.IsNullOrWhiteSpace(search))
            {
                if (useRegex)
                {
                    var pattern = search.Trim();
                    query = query.Where(p => Regex.IsMatch(p.FolderPath + "/" + p.FileName, pattern, RegexOptions.IgnoreCase));
                }
                else
                {
                    // EF emits: LOWER("FolderPath" || '/' || "FileName") LIKE @pattern ESCAPE '\'
                    // This matches the IX_Photos_PathTrgm GIN expression byte-for-byte so the
                    // planner can use the trigram index for unanchored substring search.
                    var escapedLower = EscapeLikePattern(search.Trim()).ToLowerInvariant();
                    var pattern = $"%{escapedLower}%";
                    query = query.Where(p => EF.Functions.Like(
                        (p.FolderPath + "/" + p.FileName).ToLower(),
                        pattern,
                        "\\"));
                }
            }

            if (tags != null && tags.Count > 0)
            {
                var normalizedTags = tags
                    .Where(t => !string.IsNullOrWhiteSpace(t))
                    .Select(t => t.Trim().ToLowerInvariant())
                    .Distinct()
                    .ToList();

                if (normalizedTags.Count > 0)
                {
                    var requiredCount = normalizedTags.Count;
                    var matchingPhotoIds = _context.PhotoTags
                        .Where(pt => normalizedTags.Contains(pt.Tag.Name))
                        .GroupBy(pt => pt.PhotoId)
                        .Where(g => g.Select(x => x.TagId).Distinct().Count() == requiredCount)
                        .Select(g => g.Key);

                    query = query.Where(p => matchingPhotoIds.Contains(p.Id));
                }
            }

            return query;
        }
```

```csharp
        public async Task<(List<Photo> Items, int Total)> GetPhotosAsync(
            List<string>? tags, string? search, bool useRegex, bool withoutTags, int page, int pageSize,
            CancellationToken cancellationToken)
        {
            IQueryable<Photo> baseQuery = BuildFilterQuery(tags, search, useRegex);

            if (withoutTags)
                baseQuery = baseQuery.Where(p => !p.Tags.Any());

            // Count runs on the bare filtered query — no Include, no order, no skip/take.
            var total = await baseQuery.CountAsync(cancellationToken);

            var items = await baseQuery
                .OrderByDescending(p => p.ModifiedAt)
                .ThenByDescending(p => p.Id)
                .Skip((page - 1) * pageSize)
                .Take(pageSize)
                .Include(p => p.Tags)
                    .ThenInclude(pt => pt.Tag)
                .AsSplitQuery()
                .ToListAsync(cancellationToken);

            return (items, total);
        }
```

Notes:
- `AsNoTracking()` already applies — `BuildFilterQuery` sets it on the root.
- `AsSplitQuery()` causes EF to emit two SQL statements: (1) `SELECT ... FROM Photos WHERE ... ORDER BY ModifiedAt DESC, Id DESC LIMIT N OFFSET M`; (2) `SELECT ... FROM PhotoTags JOIN PhotobankTags ... WHERE PhotoId IN (<page ids>)`. EF preserves the parent ordering when stitching results.
- `Include` is applied **after** `Skip/Take` deliberately. EF Core 8 supports this and produces the correct paged + included SQL when combined with `AsSplitQuery`.
