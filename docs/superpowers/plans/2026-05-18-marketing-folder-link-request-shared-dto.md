# Extract Shared MarketingFolderLinkRequest DTO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the cross-contract nested type `CreateMarketingActionRequest.CreateFolderLinkRequest` with a standalone shared DTO `MarketingFolderLinkRequest`, consumed by both `CreateMarketingActionRequest` and `UpdateMarketingActionRequest`, with no change to wire format or behavior.

**Architecture:** Pure type-level refactor in the Marketing module's application-layer Contracts folder. One new file (the shared DTO), three modified files (two request contracts + one test), one new baseline test for the Create-side `FolderLinks` round trip, and a build to regenerate the OpenAPI/TypeScript client. No handler logic, persistence, validation, or UI changes.

**Tech Stack:** .NET 8, C# 12 (class-based DTOs — records forbidden by project rule), MediatR, xUnit + FluentAssertions + Moq, NSwag (auto-regenerates `frontend/src/api/generated/api-client.ts` on build), React/TypeScript frontend (no hand-written FE changes expected).

            .Callback<MarketingAction, CancellationToken>((a, _) => capturedAction = a)

        capturedAction.FolderLinks.Single().FolderKey.Should().Be("key-1");
        capturedAction.FolderLinks.Single().FolderType.Should().Be(MarketingFolderType.General);

using System.ComponentModel.DataAnnotations;
using Anela.Heblo.Domain.Features.Marketing;

namespace Anela.Heblo.Application.Features.Marketing.Contracts

    public class MarketingFolderLinkRequest
