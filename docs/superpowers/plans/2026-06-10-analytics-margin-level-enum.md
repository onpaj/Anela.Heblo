# Strongly-Type `MarginLevel` in Analytics Module — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the stringly-typed `MarginLevel` parameter in the Analytics module with a domain-layer `enum`, eliminating the silent-fallback-to-M2 bug on invalid input and aligning with sibling discriminators (`ProductGroupingMode`, `AnalyticsProductType`).

**Architecture:** Add a new `Anela.Heblo.Domain.Features.Analytics.MarginLevel` enum (members `M0, M1, M2`). Propagate the enum through the request DTO, response DTO, `IMarginCalculator` (both `CalculateAsync` and `GetMarginAmountForLevel`), `IMonthlyBreakdownGenerator.Generate` plus its private helpers, and `GetProductMarginSummaryHandler`. Replace the silent fallback in `GetMarginAmountForLevel` with `ArgumentOutOfRangeException`. The wire format (`marginLevel=M0|M1|M2`) is unchanged because `JsonStringEnumConverter` is globally registered. Tests that asserted the legacy case-insensitive / silent-fallback semantics are deleted and replaced with a throw-test plus a model-binding integration test. The OpenAPI/NSwag pipeline regenerates the TypeScript client; the frontend's local `MarginLevelType` literal union is swapped for the generated string-valued enum.

**Tech Stack:** .NET 8, MediatR, ASP.NET Core controllers, xUnit + FluentAssertions + Moq, NSwag OpenAPI generation, React + TypeScript.
