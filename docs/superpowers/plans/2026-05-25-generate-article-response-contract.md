# Complete GenerateArticleResponse Contract Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend `GenerateArticleResponse` with `HangfireJobId` and `Status` so `POST /api/Articles/generate` returns the full three-field contract defined in the article-generation feature spec.

**Architecture:** Backend-only, additive change. `GenerateArticleResponse` gains two properties (`HangfireJobId`, `Status`) and `ArticleId` becomes nullable. The handler captures the `string` job ID already returned by `IBackgroundJobClient.Enqueue<T>` and assigns it onto the response with `Status = ArticleStatus.Queued`. The TypeScript OpenAPI client is regenerated by `dotnet build` and committed.

**Tech Stack:** .NET 8, MediatR, Hangfire (`IBackgroundJobClient`), xUnit, Moq, FluentAssertions, NSwag/OpenAPI client generation.

## Key Implementation Notes

1. **Moq cannot mock the `Enqueue<T>` extension method.** It delegates to `IBackgroundJobClient.Create(Job, IState)`. To set a return value, mock `Create`:
   ```csharp
   _backgroundJobClient
       .Setup(c => c.Create(It.IsAny<Job>(), It.IsAny<EnqueuedState>()))
       .Returns("job-123");
   ```
   The existing `Handle_PersistsAndEnqueuesHangfireJob` test already uses `Verify(c => c.Create(...))` — mirror that pattern for `Setup`.

2. **`ArticleId` becomes `Guid?`.** No internal C# consumer reads `.ArticleId` today (verified via grep — only `GenerateArticleHandler` writes it; the controller forwards the DTO to JSON serialization via `HandleResponse`).

3. **`Status` is non-nullable.** Enum zero value is `Queued`. The XML doc on `Status` clarifies it is meaningful only when `Success == true`. `BaseApiController.HandleResponse` returns 4xx/5xx for failure responses, so HTTP-status-aware clients never read `Status` from a failure body.

4. **Project rule:** No comments unless non-obvious. The XML doc on `Status` qualifies because the type system cannot express the "valid only on success" invariant.

5. **`dotnet format` runs on every commit.** Match existing handler style: `var jobId = ...`.
